Page 01 — Introduction

What is cterm?

A complete terminal emulator built from scratch in C, without GTK or Qt

cterm is a production-quality terminal emulator written from scratch in C99. Unlike most terminal emulators that rely on GTK or Qt for their GUI, cterm uses SDL2 directly — giving it GPU-accelerated rendering, minimal dependencies, and portability across Linux and Windows.

Every component was built by hand: the PTY process management, the ANSI/VT100 escape sequence parser, the FreeType glyph cache, the split-pane tree, the tab system, and the built-in tool launcher. This makes cterm both a working terminal and a learning resource for systems programming in C.

Why cterm?

⚡

GPU Accelerated

All text rendering uses FreeType glyph textures drawn by the SDL2 GPU renderer. Dirty-cell optimization skips unchanged cells.

🔧

Zero Framework

No GTK, no Qt, no Electron. Just SDL2 + FreeType + POSIX. Minimal dependencies, maximum portability.

🗂️

Tabs & Panes

Multiple independent shell sessions in tabs. Horizontal and vertical split panes — each with its own PTY.

🚀

Tool Launcher

Press Ctrl+P to open a searchable launcher for 60+ tools. Auto-detects what is installed on your system.

Quick Start

First Launch

When cterm opens you will see a tab bar at the top and a bash shell prompt. Everything works exactly like any Linux terminal — type commands, press Enter, use Tab completion, Ctrl+C to interrupt.

ℹ

cterm runs a real bash process connected via PTY. All shell features work: aliases, environment variables, history, job control, shell scripts.

The tab bar runs across the top of the window. Each tab represents one workspace containing one or more panes.

Split Panes

Each tab can be split into multiple panes arranged as a binary tree. Every pane has its own independent bash process — they do not share state.

ConceptDescription
Focused paneBlue border. Keyboard input goes here.
Divider4px bar between panes. Visual separator only in v1.0.

Background panesKeep running — their output is parsed even when not focused.

Scrollback

cterm keeps 5000 lines of history per pane. When you scroll up, a blue line appears at the top indicating you are in history mode. The cursor disappears. Any keypress returns you to live view.

Tool Launcher

Built-in launcher for 60+ development and security tools

Press Ctrl+P to open the tool launcher. It searches all installed tools on your system and lets you launch any of them in a new tab with a single keypress.

How auto-detection works

cterm searches every directory in your $PATH environment variable for each registered tool. Only tools that are actually installed and executable appear in the list. When you install a new tool and restart cterm, it appears automatically.

Configuration

On first launch, cterm creates a default configuration file at ~/.config/cterm/cterm.conf. Edit this file with any text editor and restart cterm to apply changes.

Architecture

How cterm works internally

High-level data flow

SDL2 Window │ ├── SDL_PollEvent() ──────────────────────────────────────────────────┐ │ │ │ │ keyboard / mouse tab bar │ │ │ │ pty_write() ──► bash process (per pane) tabs_handle_click() │ │ │ bash output │ │ │ pty_read() [non-blocking] │ │ │ terminal_process() ◄── ANSI/VT100 parser │ │ │ cell grid update (dirty=1) │ └── Render loop (60fps cap) │ pane_layout() ← compute pixel rects from pane tree │ render_pane_tree() │ ├── for each leaf pane: │ for each cell where dirty==1: │ SDL_RenderFillRect() ← background │ font_draw_char() ← glyph │ cell.dirty = 0 │ pane_draw_dividers() │ tools_launcher_draw() [if open] │ SDL_RenderPresent() ← flip to screen

Project file structure

Key data structures

Cell — one character position

typedef struct {
    char  ch;      /* printable character            */
    Color fg;      /* foreground RGB color           */
    Color bg;      /* background RGB color           */
    int   bold;    /* bold flag                      */
    int   dirty;   /* 1 = changed, needs redraw      */
} Cell;

Pane — split pane tree node

typedef struct Pane {
    PaneType       type;      /* LEAF, SPLIT_H, SPLIT_V        */
    Terminal      *term;      /* cell grid (leaf only)         */
    PTY            pty;       /* shell process (leaf only)     */
    struct Pane   *first;     /* left/top child (split only)   */
    struct Pane   *second;    /* right/bottom child            */
    float          ratio;     /* 0.5 = equal split             */
    SDL_Rect       rect;      /* pixel bounds (from layout)    */
    int            focused;   /* 1 = receives keyboard input   */
} Pane;
Page 09 — Developer Guide

Module Reference

Detailed description of every source file

main.c — Entry point & main loop

Initializes all modules in order. Runs the main loop: poll SDL events → read PTY → compute pane layout → render. Implements the 60fps frame cap using SDL_GetTicks and SDL_Delay. Handles all keyboard shortcuts and routes events to the correct subsystem.

window.c — SDL2 window management

Creates the SDL2 window with SDL_WINDOW_RESIZABLE. Manages the GPU renderer (SDL_RENDERER_ACCELERATED + SDL_RENDERER_PRESENTVSYNC). Implements F11 fullscreen toggle using SDL_WINDOW_FULLSCREEN_DESKTOP. Sets minimum window size (400×200).

font.c — FreeType glyph cache

Loads a .ttf font using FreeType. Pre-renders all 96 printable ASCII characters into SDL2 textures at startup. The glyph cache means each character is rasterized once and reused every frame. Implements font_draw_char() and font_draw_string() using SDL_SetTextureColorMod for colorization.

pty.c — Pseudo-terminal management

Calls openpty() to get a PTY pair. Forks a child process and exec's bash with TERM=xterm-256color, COLUMNS, LINES set. Sets O_NONBLOCK on the master fd. Sends SIGWINCH via a background child after 80ms to force readline to re-query terminal size. pty_resize() calls TIOCSWINSZ.

ansi.c — ANSI/VT100 parser

Implements a 4-state FSM (NORMAL, ESCAPE, CSI, OSC). Handles all standard cursor movement, erase, insert/delete, SGR colors (16-color and truecolor). OSC state silently consumes title-setting sequences. DCH and ICH implement Delete key and mid-line insert. clear_cell() resets bg+fg+bold for correct erase behavior.

tabs.c — Tab management

TabManager owns an array of Tab structs. Each Tab owns a pane tree (root Pane pointer). tabs_draw_bar() renders the tab bar using SDL2 primitives and font_draw_string(). tabs_handle_click() handles mouse clicks in the tab bar zone. Tabs shift left on close to maintain compact array.

pane.c — Split pane tree

Binary tree where leaf nodes hold Terminal+PTY and split nodes hold two children + a ratio. pane_layout() recursively computes pixel SDL_Rects. pane_split() creates a new split node replacing a leaf. pane_close_focused() collapses a split, giving space to the sibling. pane_read_all() reads PTY output for every leaf.

tools.c — Tool launcher

ToolManager registers 60+ tools. smart_register() calls tool_is_installed() which searches $PATH — only installed tools appear. Launcher overlay draws with full scroll support: scroll_offset tracks visible window, ensure_visible() adjusts it after selection changes. tools_launch() opens a new tab and sends the command to bash.

config.c — Configuration file

config_load() reads ~/.config/cterm/cterm.conf. Parser splits lines on '=', trims whitespace, matches known keys. Unknown keys silently ignored. config_save_default() creates the directory and file on first run. All modules read g_config (global Config struct) instead of hardcoded values.

Security model

cterm is designed to be safe by default:

PropertyDetail
No privilege escalationRuns entirely as the current user. Never needs root.
No network accesscterm itself makes zero network connections.
OSC sanitizationAll OSC/DCS sequences consumed silently, never executed.
Process isolationEach tab/pane is an independent process group.
Memory safetyAll malloc returns checked, bounds clamped before array access.
No execution of configfont_path is passed to FreeType only — it does not execute code.

Hardened release build

For distribution, add these flags to CMakeLists.txt:

target_compile_options(cterm PRIVATE
    -O2
    -D_FORTIFY_SOURCE=2
    -fstack-protector-strong
    -Wformat -Wformat-security
)
target_link_options(cterm PRIVATE
    -Wl,-z,relro -Wl,-z,now
)
